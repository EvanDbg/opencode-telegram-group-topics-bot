# v0.13.2

## Release Summary

This release brings the group-topics fork forward through the upstream `v0.13.2` review point while preserving thread-scoped session routing.

The biggest additions are scheduled tasks, live assistant response streaming, streamed tool-call updates, and scoped pinned-session cost tracking. In forum groups, the first scheduled task for a project now creates a dedicated scheduled topic so background automations stay separate from active coding lanes.

## Highlights

- Adds `/task` and `/tasklist` for project-scoped scheduled automations
- Creates and reuses a dedicated per-project scheduled topic in forum groups
- Streams assistant replies and tool-call updates into the correct Telegram topic by session binding
- Shows cumulative session cost in scoped pinned messages
- Adds `/commands` pagination, filtering, and clearer execution messaging

## Included Since v0.11.5

- Nested v2 scoped settings persistence and migration coverage
- Scheduled task storage, schedule parsing, execution runtime, and Telegram delivery
- Thread-aware assistant response streaming with completion fallback handling
- Tool-call streaming with topic-safe flushing around questions, permissions, files, and errors
- Markdown formatting fixes for streamed tables and richer Telegram-safe output
- Scoped pinned-message cost aggregation from live events and session history
- `/commands` page navigation and filtering for non-command prompt entries
- New regression coverage for streaming, scheduled tasks, pinned cost, and scoped settings

## Fork Notes

- Scheduled task runs use temporary OpenCode sessions and do not replace the active session bound to a topic thread
- Forum groups keep scheduled task output in a dedicated scheduled topic instead of mixing automation chatter with live coding topics
- Streamed edits stay routed by `sessionId` to the owning topic, so one busy thread does not steal another thread's output
