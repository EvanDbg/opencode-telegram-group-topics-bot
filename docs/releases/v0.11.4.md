# v0.11.4

## Release Summary

This release brings the group-topics fork forward to the upstream `v0.11.4` review point while keeping the fork's topic-scoped workflow intact.

The main runtime change is a safer Telegram text-delivery path: assistant replies still use MarkdownV2 when available, but interactive and status-oriented bot messages now avoid fragile markdown formatting and continue to fail gracefully back to plain text when Telegram rejects entities.

## Highlights

- Ports upstream Telegram markdown-parse fixes in a thread-safe way for forum topics
- Adds graceful fallback for edited bot messages as well as sent messages
- Keeps assistant formatting support while making status, permission, question, and startup flows safer in topic threads
- Adds French locale support across config, i18n helpers, and docs

## Included Since v0.11.2

- Raw-text delivery for fragile bot/system messages
- Markdown fallback support for edited Telegram messages
- New shared Telegram text helper for `raw` vs `markdown_v2` delivery
- Permission labels for `external_directory`
- French locale registration and validation
- Updated locale docs and fork sync notes for upstream `v0.11.4`

## Stability Notes

- Assistant completions still use the configured formatter and now continue more safely when Telegram rejects MarkdownV2 entities
- Group General/control-lane routing and topic-bound session lanes remain unchanged
- Thread-aware status, permission, and question flows continue to send into the correct topic thread
