# v0.13.5

## Release Summary

This patch release makes scheduled-task setup less fragile when you change defaults mid-flow or abort before a session exists.

It keeps task setup active while you open inline default pickers, applies the final agent/model/variant defaults at the moment you send the scheduled prompt, ignores reply-keyboard control taps during setup, and makes `/abort` cleanly cancel setup-only interactions.

## Highlights

- Lets scheduled-task setup stay open while you change agent, model, or variant defaults before the final prompt
- Uses the latest active defaults when the final scheduled-task prompt is submitted
- Prevents reply-keyboard control taps from being mistaken for task-setup text input
- Makes `/abort` cancel setup-only interactions even when no OpenCode session is running
- Documents the permanent upstream fork-sync remote for future maintenance work

## Notes

- Default changes remain limited to agent, model, and variant once the scheduled-task prompt stage is active
- Inline menus now restore the suspended task prompt state after they close
- Fork-sync guidance now points maintainers to the permanent `upstream` remote and `FORK_SYNC.md`

## Included Tests

- Task setup keeps running while inline default menus are opened and closed
- Final scheduled-task submission uses the latest stored defaults
- `/abort` clears setup-only task interactions without requiring an active session
- Menu-blocking messaging and reply-keyboard control behavior during task setup
