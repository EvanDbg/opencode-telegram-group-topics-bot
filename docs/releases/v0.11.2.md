# v0.11.2

## Release Summary

First public release of the group-topics fork.

This release takes the original single-chat OpenCode Telegram bot and turns it into a group-first workflow built around Telegram forum topics. One group can now act as a project workspace, with a shared control lane in **General** and dedicated topics for parallel OpenCode sessions.

It also brings the fork up to date with selected upstream improvements where they help this group-scoped model, while keeping the fork's thread-scoped behavior as the source of truth.

## Highlights

- Group topics become first-class session lanes for OpenCode work
- Multiple topics can run in parallel inside one group
- Multiple groups can be active in parallel across different projects
- New onboarding and setup flow now works end-to-end for published installs
- Fork documentation now clearly explains how this project differs from the upstream single-chat bot

## What Makes This Release Different

- The upstream project is optimized for a simpler single-chat experience
- This fork is optimized for users who want a more powerful multi-session mobile workflow
- That added power comes with more moving parts, but the payoff is a much better parallel workflow for active OpenCode usage from Telegram

## Stability Notes

- OpenCode sessions continue running even when Telegram update frequency slows down under heavy parallel usage
- Telegram rate limits are handled gracefully, so high activity may reduce update frequency without breaking the underlying session work
- The setup wizard, config flow, and published install path have now been exercised in production
